# Baton todo creation

Create project-native GitHub issues that remain useful without Baton. Use the
project's normal issue workflow and structure.

For each independent outcome:

1. Split unrelated outcomes into separate issues.
2. Write a durable problem or desired outcome, relevant evidence, constraints,
   and observable acceptance criteria.
3. Create the issue through the project's normal GitHub workflow and retain
   its returned number and URL.

Complete issue creation when every independent outcome has one recorded issue.
