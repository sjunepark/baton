# Baton v0.4.2 Adopter Update

Adopting repo action: Optional runtime bump.

Runtime:

- Update the Baton install pin in `.github/workflows/*` to `v0.4.2` when the
  repo wants the latest fixes.

Config:

- No config migration is required.
- New Baton installs include `setup.baseline_baton_version: v0.4.2` in
  `.github/baton.yml`.
- Existing repos may add that marker when they review their setup against this
  release.

Templates:

- Run `baton init --dry-run --json` and review the plan before applying any
  template updates.

Labels:

- Run `baton sync-labels --dry-run --repo owner/name --json` and review the
  label diff before applying.

Checks:

```sh
baton init --dry-run --json
baton migrate-config --dry-run
baton sync-labels --dry-run --repo owner/name --json
baton ensure-branch --json
baton doctor --format toon
```
