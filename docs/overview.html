Baton Overview

Baton is a local-first CLI and Codex skill for reusable GitHub issue and PR agent workflows. It owns deterministic policy, queue, branch, and completion facts. Checkout isolation belongs to the caller.

Boundary

Baton does not create, lease, release, prune, or delete worktrees. Coda, Treehouse, Codex background checkouts, or the user provide the isolated execution context before implementation or PR follow-up edits happen.

Core Flow

baton home --format toon
baton next --format toon --repo owner/name
# choose exactly one candidate
# edit only inside a caller-provided isolated checkout
baton complete --summary "..." --validation "..." --json
