Baton Tutorial

Use Baton to inspect policy-safe GitHub work, choose one candidate, and record the result. Bring your own isolated checkout before editing.

Inspect

baton home --format toon
baton doctor --format toon
baton queue --format toon --repo owner/name
baton next --format toon --repo owner/name

Implement One Issue

# In a caller-provided isolated checkout:
git switch -c agent-work/123-short-slug origin/agent
# implement, validate, push, and open/update the PR
baton complete --summary "Implemented issue 123" --validation "go test ./..." --json

Follow Up One PR

baton pr 42 --json --repo owner/name
baton checks 42 --format toon --repo owner/name
baton review-threads 42 --format toon --repo owner/name
# In a caller-provided isolated checkout:
git switch <head-ref>
# fix, validate, and push to the same branch
baton complete --summary "Handled PR 42 follow-up" --validation "go test ./..." --json

Rule

If the current directory is the user's primary checkout or isolation is uncertain, stop and report the missing execution context. Baton will not create or clean up a worktree for you.
